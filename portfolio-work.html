  • Sujet: Schématisation des flux entre les applications métier
  • Durée: 4 mois
  • Besoin: Le but est de créer un rapport Power BI qui consiste à cartographier les différents flux entre toutes les applications métier de l’entreprise afin d’optimiser l’interconnexion entre ces dernières. Cette cartographie facilitera la migration des applications et des interfaces vers Azure et identifiera les impacts.

Déroulement du projet :

  • Phase 1 - Étude préliminaire du besoin et de l’existant : Dans cette phase, nous avons premièrement analysé l’existant. Au sein de l’entreprise, nous avons un référentiel applicatif et un référentiel des interfaces qui étaient tous deux obsolètes ; nous avons donc besoin de faire des inventaires dans le but de mettre à jour ces référentiels. Nous avons fait un recueil d’informations auprès des différents responsables d’application pour mettre à jour le nouveau référentiel. Deuxièmement, nous avons identifié les entités circulant dans les flux de données applicatifs, les fréquences, les applications sources et cibles. Les entités ayant leurs propres propriétés, nous avons mis en place un dictionnaire de données afin de créer des entités référentielles qui pointent sur ses différents synonymes afin de les normaliser.
  • Phase 2 - Réalisation du projet : Nous avons modélisé une base de données et nous l’avons peuplée du référentiel applicatifs, le référentiel des interfaces et le dictionnaire de données (entités). De plus, nous avons créé une application Power Apps qui pointe directement sur la base de données dans le but de l’utiliser pour modifier, mettre à jour le référentiel et supprimer des entités. Cette application représente l’IHM qui fait le lien entre l’utilisateur (responsables d’application/interface) et la base de données. D’autre part, nous avons élaboré un rapport Power BI, dans lequel on retrouve toutes les interfaces et entités mais aussi pour voir la circulation des données (entités) dans le système d’information de l’entreprise afin d’optimiser le circuit de données.
  • Phase 3 - Mise en place : Nous avons conçu une application Power BI, dans laquelle nous avons identifié l’audience puis nous y avons rajouté le rapport Power BI préalablement élaboré afin qu’il soit accessible à toutes l’audience concernée.
