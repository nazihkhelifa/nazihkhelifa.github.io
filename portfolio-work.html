  • Sujet:Diagram of flows between business applications
  • Duration: 4 months
  • Need:The objective is to create a Power BI report that maps the various flows between all the company's business applications in order to optimise the interconnection between them. This mapping will facilitate the migration of applications and interfaces to Azure and identify the impacts.

Project progress :

  • Phase 1 - Preliminary study of needs and existing situation : In this phase, we first analysed the existing situation. Within the company, we had an application repository and an interface repository, both of which were obsolete; we therefore needed to make inventories with a view to updating these repositories. We gathered information from the various application managers in order to update the new repository. Secondly, we identified the entities circulating in the application data flows, the frequencies, and the source and target applications. As the entities have their own properties, we set up a data dictionary to create referential entities that point to their various synonyms in order to standardise them.
  • Phase 2 - Project implementation: We modelled a database and populated it with the application repository, the interface repository and the data dictionary (entities). In addition, we created a Power Apps application that points directly to the database in order to use it to modify, update the repository and delete entities. This application represents the GUI that links the user (application/interface managers) to the database. In addition, we have developed a Power BI report, in which all the interfaces and entities can be found, but also to see the circulation of data (entities) in the company's information system in order to optimise the data circuit.
  • Phase 3 - Implementation: We designed a Power BI application in which we identified the audience and then added the Power BI report we had previously developed so that it was accessible to all the audience concerned.
